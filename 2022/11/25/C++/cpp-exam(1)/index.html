题目三

1. 友元引出

首先需要明确几个访问权限的问题

一、组合类无法访问其组件类内部的私有成员变量,如:

Rectangle::~Rectangle() {
	p1.getX();//无法访问p1.x
	counter--;
}

二、相同类可以访问相同类的其他对象的私有成员变量,如:

double getPx(Point& p2) {
	cout << p2.x << endl;
}

三、成员函数可以访问其类中私有成员,但如果私有成员是对象类型,而且非同类型的对象,则无法访问私有成员的私有成员。

友元关系的出现打破了三的约束,友元关系提供了不同类或者对象的成员函数之间、类的成员函数与一般函数之间进行数据共享的机制。简单地说,友元关系就是一个类主动声明哪些其他类或者函数是它的朋友,进而给它们提供对本类的访问特许,也就是说,通过友元关系,一个普通函数或者类的成员函数可以访问封装于另一个类中的数据。

在一个类中，可以利用关键字friend将其他函数或者类声明为友元，如果友元是一般函数或者类的成员函数,那么就称为友元函数,如果友元是一个类,那么就称为友元类,友元类的所有成员函数都自动成为友元函数

2. 友元函数

友元函数在类中用关键字friend修饰的非成员函数,友元函数可以是一个普通的函数,也可以是其他类的成员函数。虽然它不是本类的成员函数,但是在它的函数体中可以通过对象名访问类的私有和保护成员。

3. 友元类

同友元函数一样,一个类可将另一个类声明为友元类,如果A类是B类的友元类,那么A类中的所有成员函数都是B的友元函数，都能够访问B类的私有和保护成员

  • 友元关系是不能够传递的,B类是A类的友元,C类是B类的友元,C类和A类之间,如果没有声明,就没有任何友元关系。
  • 友元关系是单向的,如果声明B类是A类的友元,B类的成员函数就可以访问A类的私有和保护数据。
  • 友元关系是不被继承的,如果类B是类A的友元,类B的派生类不会自动称为类A的友元
#include <iostream>

using namespace std;
#define PI 3.14159
class Point {
public:
	Point(Point& point) {
		this->x = point.x;
		this->y = point.y;
	}
	Point(double x, double y) {
		this->x = x;
		this->y = y;
	}
	void setX(double x) {
		this->x = x;
	}
	double getX() {
		return this->x;
	}
	void setY(double y) {
		this->y = y;
	}
	double getY() {
		return this->y;
	}
private:
	double x;
	double y;
};

class Circle {
public:
	Circle(Point& center,double radius) :center(center),radius(radius) {};
	double getArea() {
		return PI * radius * radius;
	}
	friend double calArea(Circle& c);
private:
	double radius;
	Point center;
};

double calArea(Circle& c) {
	return PI * c.radius * c.radius;
}

int main() {
	Point center(2, 2);
	Circle c(center,2);
	cout << "Radius：" << 2 << ", PI = " << 3.14159 << endl;
	cout << "成员函数GetArea()计算 The area is : " << c.getArea() << endl;
	cout << "友元函数calArea()计算 The area is : " << calArea(c) << endl;
	return 0;
}

实验三

题目一

对于既需要共享又需要放置改变的数据应该声明为常量,因为常量在程序运行期间是不可以改变的,所以可以有效地保护数据。

1. 常对象

常对象:它的数据成员值在对象的整个生存期间内部能被改变,常对象必须进行初始化,而且不能被更新

//声明常对象的语法形式为:
const 类型说明符 对象名;

在声明常对象的时候把const关键字放在类型名后面也是允许的。

与基本数据类型的常量相似,常对象的值也是不能被修改的

语法保障类类型的常对象的值不能被改变,改变对象的数据成员值有两个途径

  • 通过访问对象名访问其成员对象,由于常对象的数据成员都被视同为常量,这时候语法限定其不能被赋值
  • 在类的成员函数中改变数据成员的值,然而几乎无法预料和统计哪些成员函数可能会改变数据成员的值,因此语法规定的是 不能通过常对象调用普通的成员函数

2. 用const修饰的类成员

  • 常成员函数

使用const关键字修饰的函数是常成员函数, 常成员函数声明格式如

类型说明符 函数名(参数表)const;

(1)const是函数类型的一个组成部分,因此在函数的定义部分也要带const关键字

(2)如果将一个对象声明为常对象,则通过该常对象就只能调用它的常成员函数,而无法调用其他成员函数

(3)无论是否通过常对象调用常成员函数,在常成员函数调用期间,调用这个函数的对象都将被看作是常对象,因此常成员函数不能更新目的对象的数据成员,也不能针对目的对象调用该类中没有用const修饰的成员函数。

(4)const关键字可以用于对重载函数的区分

void print();
void print()const;

这是对print()的有效重载,

如果仅以const关键字为区分对成员函数的重载,那么通过非const的对象调用该函数,两个重载的函数都可以与之匹配,这时候编译器就会选择最近的函数不带const关键字的函数

  • 常数据成员

类的成员数据也可以是常量,使用const说明的数据成员为常数据成员, 如果在一个类中说明了常数据成员,那么任何函数中都不能对该成员赋值,构造函数对该数据成员进行初始化,就只能通过初始化列表

class A {
public:
	A(int a);
	const int a;
	static const int b;
};

//对于静态的常数据成员,可以在类外说明和做初始化
const int A::b = 10;

//常数据成员只能通过初始化列表来获得初值 
A::A(int a):a(a){

}

类成员中的静态变量和常量都应该在类体之外加以定义,但是C++标准规定了一个例外:

类的静态常量如果具有整数类型或者枚举类型,那么可以直接类定义为它指定常量值

static const int b = 10;

这时候不必再在类的定义外定义A::b,因为编译器会将程序中对A::b的所有引用都替换为数值10,这实时无需为A::b分配空间

但是如果程序中出现了对b取地址的情况,则必须通过专门的定义为A::b分配空间,由于已经在类定义中为它指定了初值,不能再在类定义外为它指定初值,即使两处给出的初值相同也不行。

3. 常引用

如果在声明引用的时候用const修饰,被声明的引用就是常引用,使用常引用所引用的对象不能被更新,如果常引用做形参,不会意外地发生对实参的修改

const 类型说明符 &引用名;

非const的引用只能绑定到普通的对象而无法绑定到常对象,但是常引用可以绑定到常对象。一个常引用,无法绑定到一个普通对象还是一个常对象,通过该引用访问该对象的时候,都只能够把这个对象当做常对象,这意味着,对于基本数据类型的引用,不能为数据赋值,对于类类型的引用,不能修改数据成员,不能调用非常成员函数的成员函数

#include <iostream>
using namespace std;

class Point {
public:

	Point(int x=1, int y=1) :x(x), y(y) {};
	int getX() const{//必须声明为常对象,无法常对象无法进行引用
		return this->x;
	}
	int getY() const{
		return this->y;
	}
	double Dist(Point p) {
		int tx = x - p.x;
		int ty = y - p.y;
		return sqrt(tx * tx + ty * ty);
	}
	double Dist(const Point p)const {
		int tx = x - p.x;
		int ty = y - p.y;
		return sqrt(tx * tx + ty * ty);
	}
private:
	int x;
	int y;
};


int main() {
	const Point myp1(1, 1), myp2(4, 5);
	Point p1;
	Point p2;
	cout<<myp1.Dist(myp2)<<endl;
	cout<<p1.Dist(p2)<<endl;
}

		score_nor[i].showScore();
	}

	for (int i = 0; i < 3; i++) {

	}

}

实验四

题目一

（1） 根据程序的运行结果，推出系统默认的拷贝构造函数是如何编写的？

CA(const CA& C){//执行的是浅拷贝,也就是说对成员数据进行简单的复制
    this->a = C.a;
    this->str=C.str;
}

（2） 分析对象A的str和对象B的str内存地址相同的原因。

这是因为原程序在类中并没有定义自己的拷贝构造函数,因此调用的是编译器提供的默认的拷贝构造函数,而系统默认的拷贝构造函数是浅拷贝,也就是把成员数据的值进行简单的复制,对指针类型的变量str,执行的是str = C.str,在这种情况下,编译器在构造B对象的时候,传递给B对象str的是A对象的str地址,因此二者的地址是相同的

（3） 上述程序实现的是浅复制还是深复制？简述两者的区别。

实现的是浅复制,因为这个程序做的只是把成员数据的值进行简单的复制,而没有在B专属的内存中,将A的数据内容在B专属的内存空间中克隆一份为B所专用。

区别

  • 浅复制:被复制的对象的所有变量都含有与原来对象相同的值,而所有其他对象的引用依然指向原来的对象。
  • 深复制:被复制对象的所有变量都含有与原来对象相同的值,那些引用其他对象的变量将指向被复制的新对象,而不是原有的那些被引用的对象

（4） 浅复制在哪种情况下会带来哪些问题？应该如果解决？

浅复制在构造含有需要分配内存空间的成员变量的对象的时候,有可能会造成内存泄漏，指针悬挂的问题。容易造成一个内存地址被释放两次的情况发生

构造这样的对象的时候,需要定义自己的拷贝构造函数,为新对象申请新的空间,并将原对象的值一一复制到新对象中。

（5） 上述程序会造成内存泄漏吗？如果存在，请分析内存泄漏会在哪一步发生？

不会造成内存泄漏,会造成指针悬挂,编译器执行的默认拷贝构造函数

这一步是在A执行完析构函数之后发生的,由于A析构了,A中的str指针所指向的内存空间也跟着释放,这时候,B中的str指针所指向的区域就是一片已经被释放了内存的区域,这时候就产生了指针悬挂现象,在后续的B执行析构函数的时候,还会出现已经被释放的内存区域被再次释放的情况出现。

题目二

（1） 上述程序的本意是统计矩形的个数，但从实际运行结果看，并没有达到这个目的，请分析原因。

这是因为原程序没有定义自己的拷贝构造函数,在执行Rect rect2(rect1)的时候,调用的是系统默认的拷贝构造函数,而系统的默认拷贝构造函数也只是对非静态成员进行简单的复制,也就是说并没有执行counter++的这个逻辑,并不会对统计对象个数的静态成员counter做任何的处理,所以,counter依然保持拷贝构造函数之前的值,没有随着对象的增加而增加。

修改上述程序，使其能够正确输出已经创建的矩形数量。请给出源代码完整程序，并粘贴运行结果截图。

#include <iostream>
using namespace std;

class Shape
{
public:
	Shape() {}
	~Shape() {}
	virtual float getArea() { return -1; }
};

class Circle : public Shape
	//定义Circle类
{
public:
	Circle(double radius) :radius(radius) {};
	~Circle(){}
	float getArea() {
		return 3.14 * radius * radius;
	}
private:
	float radius;
};

class Rectangle : public Shape
	//定义Rectangle类
{
public:
	Rectangle(int width, int height) :width(width), height(height) {};
	~Rectangle() {};
	float getArea() {
		return width * height;
	}
	float getWidth() { return width; }
	float getHeight() { return height; }
private:
	float width;
	float height;
};

class Square : public Rectangle
	//定义Aquare类
{
public:
	Square(float len):Rectangle(len,len) {};
	~Square() {};
private:
	float len;
};

int main()
{
	Shape* sp;
	sp = new Circle(5);
	cout << "The area of the Circle is " << sp->getArea() << endl;
	delete sp;
	sp = new Rectangle(4, 6);
	cout << "The area of the Rectangle is " << sp->getArea() << endl;
	delete sp;
	sp = new Square(5);
	cout << "The area of the Square is " << sp->getArea() << endl;
	delete sp;
	return 0;
}
