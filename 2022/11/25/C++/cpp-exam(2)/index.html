实验五

复习:关于类的派生与继承

在C++中,派生类的一般定义语法为

class 派生类名:继承方式 基类名,继承方式 基类名,...,继承方式 基类名{
    派生类成员声明;
}

一个派生类可以同时有多个基类,这种情况称为多继承,而一个派生类只有一个直接基类的情况就称为单继承。

继承方式规定了如何访问从基类的成员,在派生类定义的语句中,每个继承的语句只限定紧随其后的基类。继承方式的关键字为public、protected、private分别表示私有继承、保护继承和私有继承。

如果不显式地给出关键字,那么系统的默认值就是私有继承。

派生类成员是指出了从基类继承来的所有成员外,新增加的数据和函数成员。

1. 派生类的生成过程

经历了3个步骤:吸收基类成员、改造基类成员、添加新的成员。

  • 吸收基类成员

第一步是将基类的成员全盘接收,派生类实际上就包含它的全部基类中除构造和析构函数之外的所有成员。

  • 改造基类成员

基类成员的访问控制是依靠派生类定义时的继承方式来控制的。

另一个对基类数据或者函数成员的覆盖或者隐藏,隐藏就是在派生类中声明一个和基类数据或者函数同名的成员,如果派生类中声明了一个和某基类成员同名的新成员,派生的新成员就隐藏了外层的同名成员。这时候再派生类中或者通过派生类的对象直接使用成员名就只能访问到派生类中声明的同名成员,这就叫做同名隐藏。

  • 添加新的成员

由于在派生过程中,基类的构造函数和析构函数是不能够被继承的,因此要实现一些特别的初始化和扫尾工作。

2. 访问控制

  • 公有继承

当类的继承方式为公有继承的时候,基类的公有和保护成员的访问属性在派生类中不变,而基类的私有成员不可直接访问。

在公有继承的前提下,从基类继承过来的公有成员和保护成员访问属性不变,而无论是派生类的成员还是派生类的对象都无法直接访问基类的私有成员。

  • 私有继承

当类的继承方式是私有继承的时候,基类中的公有成员和保护成员都以私有成员的身份出现在派生类中

  • 保护继承

保护继承中,基类的公有和保护成员都以保护成员的身份出现在派生类之中,而基类的私有成员不可以直接访问。

3. 类型兼容规则

类型兼容规则是指在需要基类对象的任何地方,都可以使用派生类的对象来替代。

  • 派生类的对象可以隐含转换为基类对象
  • 派生类的对象可以初始化基类的引用
  • 派生类的指针可以隐含转换为基类的指针

派生类对象可以隐含转换为基类对象,也就是能用派生类对象中从基类继承来的成员,逐个赋值给基类对象的成员

class B{};
class D:public B{};
B b1;
D d1;
b1 = d1;

派生类的对象也可以初始化基类对象的引用

B &rb = d1;

派生类对象的地址也可以隐含转换为指向基类的指针

B *pb1;
pb1 = &d1;

当函数的形参为基类的对象(引用、指针)的时候,实参可以是派生类的对象指针,而没有必要为每一个派生类单独设计一个模块。

4. 派生类的构造和析构函数

  • 构造函数

派生类地成员对象是由所有基类的成员对象与派生类新增的成员对象共同组成的,因此在构造派生类的对象的时候,就要对基类的成员对象和新增成员对象进行初始化。

基类的构造函数并没有继承下来,如果要完成这些工作,就必须要给派生类添加新的构造函数,为了要完成基类成员对象的初始化工作,需要通过调用基类的构造函数

在构造派生类的对象的时候,它会先调用基类的构造函数,来初始化它们的数据成员,然后按照构造函数初始化列表中指定的方式初始化派生类新增的成员对象,最后才执行派生类构造函数的函数体。

派生类名::派生类名(参数表):基类名(基类1初始化参数表),...,成员对象名1(成员对象初始参数表),...,基本类型成员初始化{
    派生类构造函数的其他初始化操作；
}

当一个类同时有多个基类的时候,对于所有需要给予参数进行初始化的基类,都要显式给出基类名和参数表,对于使用默认构造函数的基类,可以不给出基类名,同样对于成员对象,如果是使用的是默认构造函数,也不需要写出对象名和参数表。

如果对基类进行初始化的时候,需要调用基类的带有形参表的构造函数的时候,派生类就必须要声明构造函数,提供一个将参数传递给基类构造函数的途径,保证在基类进行初始化的时候能够获得必要的数据。

派生类构造函数的执行顺序如下:

  • 调用基类的构造函数,调用顺序按照它们被继承时声明的顺序从左到右
  • 对派生类新增的成员进行初始化,初始化顺序按照它们在类中声明的顺序
  • 执行派生类的构造函数体中的内容。

构造函数初始化列表中的基类名,对象名之间的顺序无关紧要,它们各自出现的顺序可以是任意的。真正的决定因素是在声明继承的时候的顺序。

如何重用直接基类定义的构造函数

class Derived:public Base{
public:
    using Base::Base;//using声明语句只是令某个名字在当前作用域内可见
    double d;
}

using声明语句将会使得编译器产生如下代码

class Derived(params):Base(args){}

当一个基类构造函数含有默认实参的时候,派生类将会获得多个继承的构造函数,其中每个构造函数分别省略掉一个含有默认实参的构造函数。

多数情况下,基类含有几个构造函数,派生类会继承所有这些构造函数,但是有两个例外

  • 派生类可以继承一部分的构造函数,为其他构造函数重新定义自己的版本,如果派生类定义的构造函数与基类的构造函数具有相同的参数列表,则该构造函数将不会被继承定义在派生类中的构造函数将替换继承而来的构造函数
  • 复制和移动构造函数不会被继承,继承的构造函数不会作为用户定义的构造函数来使用,因此一个类只含有继承的构造函数,则它也将拥有一个合成的默认的构造函数

注意:这里的继承不是真的教程,书上讲的是这样的,使用using语句进行构造函数的重用和定义并非以常规方式进行继承,但是为了方便,姑且称之为继承的

  • 复制构造函数

编译系统会在必要时自动生成一个隐含的复制构造函数,这个隐含的复制构造函数会自动调用基类的复制构造函数,一般需要为基类相应的复制构造函数传递参数。

Derived::Derived(const Derived &v):Base(v){...}
  • 析构函数

在派生过程中基类的析构函数也不能继承下来,如果需要析构,就要在派生类中声明新的析构函数,派生类析构函数的声明方法与没有继承关系的类中析构函数的声明方法完全相同。

它的执行次序如下

  • 先去执行析构函数的函数体
  • 对派生类新增的类类型的成员进行清理
  • 对所有从基类继承来的成员进行清理。

生成的析构函数体是空的,但并非不做任何事情,它会隐含地调用派生类对象成员所在类的析构函数和调用基类的析构函数。

  • 删除delete构造函数

可以通过delete来实现禁止默认构造函数或者删除复制构造函数以阻止复制的做法,在基类中删除掉的构造函数,在派生类中也是删除状态。即如果基类中的默认构造函数、复制构造函数、移动构造函数是删除或者不可访问,则派生类中对应的成员函数也将会是删除的,因为编译器无法执行派生类对象中基类部分的构造或者赋值操作。

Base(Base &)=delete;//删除复制构造函数
Base(Base &&)=delete;//删除移动构造函数

移动构造函数

复制构造函数通过复制的方式构造新的对象,而很多时候被复制的对象仅做复制用置换就被销毁。

在这时候如果使用已有对象而非复制对象将大大提高性能。

参数引用传递中对持久存在变量的引用,称为左值引用,在内存中有确定存储地址,有变量名,表示结束依然存在的值。

右值是赋值语句右侧的值,不会依附于对象(相对的对短暂存在可被移动的右值引用称为右值引用),也称为将亡值

float n = 6;
float &lr_n =n; //n是一个非常量左值,可以被非常量左值的引用绑定
float &&rr_n = n;//通过&&来获得右值引用,n是一个非常量左值,不能将右值引用绑定到左值n上
float &&rr_n = n*n;//n*n这个值随着表达式的结束而结束,属于将亡值,因此正确

可以用move函数将左值对象变为右值

float n= 10;
float &&rr_n = move(n);

4. 作用域辨析符

对于在不同作用域声明的标识符,可见性的原则是:如果存在两个或者多个具有包含关系的作用域,外层声明了一个标识符,而内层没有再次声明同名标识符,那么外层标识符在内层依然是可见的,如果在内层声明了同名标识符,则外层标识在内层是不可见的。这时候我们称内层标识符隐藏了外层的同名标识符

如果派生类中声明了与基类成员函数同名的新函数,即使函数的参数表是不同的,从基类继承的同名函数的所有重载形式也会被隐藏。

如果子类中定义的函数与父类的函数同名但是具有不同的参数表,不属于函数重载,这时候子类中的函数将隐藏父类中的同名函数,调用父类中的同名函数必须使用父类名称来限定。只有在相同的作用域中定义的函数才可以进行重载。

class Derived:public Base1,public Base2{
public:
    using Base1::var;
    using Base1::fun;
}

这样的话就可以明确你要使用哪一份成员了。
