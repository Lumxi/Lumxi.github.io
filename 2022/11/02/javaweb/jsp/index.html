JSP原理解析

1. Servlet的基本问题以及解决思路

Servlet对页面的输出控制等都是通过out.print()输出流进行打印的,这是一种耦合度非常高的开发方式。我们有一个思路,能不能生成一段功能代码,将html/css/js代码输出到out.print(),让机器将我们写的前端代码自动翻译生成Servlet这种java程序,然后机器自动将这段java编译生成class文件,再让jvm去加载这些class

那么其实可以看到了,其实本质的底层实现还是没有在变的,只是方便了程序员进行了代码编写。

简单来说就是index.jsp会被tomcat翻译生成index_jsp.java文件,然后tomcat服务器又会将index_jsp.java生成.class文件,然后交给jvm进行类加载与运行。

class HttpJspBase extends HttpServlet{}

同时可以知道JspBase也是继承自HttpServlet的,因此其本质还是Servlet

因此jsp的生命周期和Servlet的生命周期是完全一样的

因此jsp是一个(假)单例

为什么jsp文件第一次访问的时候比较慢?

这是因为第一次访问的时候类还没有生成,因此需要翻译生成jsp.java文件,然后还要编译成jsp.class文件,还要还要拿去jvm加载使用。这个过程还是比较耗时的

3. JSP是什么?

JSP是java程序，其本质还是Servlet

基于Java语言实现的服务端的页面。

JSP是一套规范,所有的web/web服务都是遵循这套规则的,都是按照这套规范进行的代码生成

每一个web容器/web服务器都内置一个JSP的翻译引擎。

E-L表达式

1. 什么是E-L表达式

Expression Language:表达式语言

E-L表达式可以代替JSP中的java代码,让JSP文件中的程序看起来更加整洁与美观

作用:从某个作用域中取数据,然后将其转化为字符串,然后将其输出到浏览器,这就是EL表达式的作用。

  • 从某个域中取数据
    • pageContext
    • request
    • session
    • application
  • 取出的数据转为字符串
    • 如果是java对象,那么则会转为toString()后输出
  • 字符串输出到浏览器。
