1. 说说你对docker的理解

https://juejin.cn/post/6844903518042603533

https://book.itxueyuan.com/XraJ/do4L

这道题可以从docker的核心概念说起,docker的核心概念有容器、镜像

从应用的层面上来讲,我们每次使用docker,都是在打包镜像,运行容器,那么在这个过程中,其实到底做了什么?我们知道容器它实际上是按照镜像的信息创建出来的,那么容器为什么能够运行呢?这就是因为镜像中设定了容器运行所需要的一套环境,包括有软件资源,代码资源,系统工具,系统库和相关的设置,因为软件运行的环境已经全部准备好了,只需要安装docker引擎,就可以在这个引擎上运行相关的软件

那么在项目中就是通过创建若干个centos容器,来模拟这样的一个操作系统,然后回显给用户

Docker镜像就是个文件夹,Docker引擎通过部署这些文件夹,然后运行相应的代码,从而创建容器

那么容器和虚拟机有什么区别呢?

对于容器,它实际上是共用操作系统内核的,因此它不需要自己模拟出来一套内核程序,而虚拟机因为需要虚拟出来一整套完备的硬件和内核程序,因此这样来说,容器它是更轻量级的,因为它不会需要再去维护一套内核程序,我们知道内核程序的运行,实际上是需要使用一定的CPU和内存的,如果是这样的话就意味着每一台虚拟机都需要使额外的系统来维护自己的内核程序

而对于使用容器技术的系统来说,由于免去了多个内核的开销,因此它是更轻量级的

但是从隔离性的角度上来讲,由于多个容器是共享同一个内核资源的,那么就可能会导致一个容器占用过多资源而导致其他容器无法运行的情况,那么这样来说,就需要通过合理的调度这些容器,管理这些容器的负载来实现了,具体的做法可以是通过docker run -mm这样的参数来限制容器对硬件的使用情况

也可以人为干预,在容器没有崩溃之前,实时检测容器的负载,如果容器占用总的负载超过了一定的比例,那么就向业务层报警,或者向数据库写入日志

docker底层是如何实现容器隔离的?

Docker之所以具有这样的优势,和它的操作系统虚拟化自身的特点是分不开的,传统的虚拟机技术需要有额外的虚拟机管理程序和虚拟机操作系统,而Docker容器是直接在操作系统层面上实现的虚拟化

  • 虚拟机和宿主机实现了严格的隔离,也就是说不管虚拟机怎么样,你的宿主机都不会发生什么,这是因为虚拟机会虚拟出来一套完整的操作系统和一套完整的硬件,这一套硬件是从实际的物理机上分离出去的,这也就意味着,虚拟机所分配的内存,磁盘,交换空间等硬件设备虽然从物理机上看是一体的,但是这些硬件设备是相互隔离的

容器和容器之间的隔离是基于namespace来实现的,它是内核中提供的一个功能,Docker提供了一个打包和运行应用的隔离环境之中,称之为容器,docker容器本质就是将系统中的同一个业务目标服务的相关进程合成一个组,放在一个叫做namespace的空间中,同一个namespace中的进程能够相互通信,但是看不到其他namespace中的进程,每个namespace可以拥有自己独立的主机名,进程ID系统,IPC,网络,文件系统,用户等等资源,Docker的隔离和安全特性允许你在一个主机上同时运行多个容器,但是它不像虚拟机那样重量级,容器都是基于宿主机的内核运行的,它底层所调用的资源的都是同一个内核提供的,这样的话就会有一个风险,就是说因为内核资源是共享,因此在一定的情况下,就会导致你的内核资源被耗尽了,其他容器无法调用内核的资源

而虚拟机的运行本质上是依赖各自的内核运行,有一种情况,就是虚拟机将自己的资源耗尽了,但是宿主机依然安然无恙

  • 那么为什么Docker要比VM要快呢?

这是因为docker有着比虚拟机更少的抽象层,它不需要通过虚拟机软件抽象出来一套自己的操作系统,不需要将硬件资源虚拟化,运行在Docker上的程序直接使用的都是虚拟机的硬件资源,因此在CPU、内存利用率上docker将在效率上有明显的优势

docker使用的是宿主机的内核,它不需要在启动的时候再去加载一个操作系统的内核,当新建一个虚拟机的时候,虚拟机软件需要加载客户机操作系统,这个新建分钟级的

namespace是什么?

namespace机制提供的是一种资源隔离的方案,PID、IPC、Network等系统资源不再是全局性的,而是属于某个特定的namespace,什么意思呢?就是说,每一个namespace有一套自己的PID列表,有一套自己的Network系统,比如说TCP/IP协议栈,这样的话,每一个namespce就只能够看到属于自己的namespace中的PID等信息,实现了每个容器的资源相互隔离,从而保障了容器内部只能够访问到自己的Namespace的资源

CGroup是什么?

  • task:任务,对应于系统中运行的一个实体,下面称为是进程
  • subsystem:子系统,具体的资源控制器,控制某个特定的资源使用
  • cgroup:控制组,一组任务和子系统的关联关系,表示对这些任务进行怎样的资源管理策略
  • hierarchy:层级树

Docker通过Cgroup来控制容器的使用的资源配额,包括有CPU、内存、磁盘三大方案,Cgroup的内核通过hook钩子来实现管理进程资源,提供了一个统一的接口,从单个进程的资源控制操作到操作系统层面的虚拟卡的过渡,它的功能主要有

  • 资源控制,通过这个CGroup来控制容器使用的资源配额
  • 优先级分配,使用硬件的权重值
  • 资源统计,使用硬件资源的用量
  • 进程控制,挂起,恢复

它是怎么和docker扯到一块的呢?这是因为docker容器本质上就一组进程,操作系统能够使得进程之间相互隔离,但是我们的目的是让这些在同一个namespace

每一个Cgroup都是一组被相同的标准和参数限制的进程,当我们启动一个容器的时候,Docker会为这个容器创建一个容器标识符相同的CGroup,每一个CGroup下面都有一个tasks文件,其中存储着属于当前控制组的所有进程的pid,作为负责CPU的子系统,当我们使用Docker关闭掉正在运行的容器的时候,Docker的子控制组对应的文件夹就会被Docker进程移除,Docker在使用CGroup其实也只是做了一些改变文件的内容,通过CGroup技术就能够解决多个容器互相抢占资源的问题

Docker的基本架构是怎么样的?

DockerClient:提供API,供外部访问

DockerDedmon:后台处理线程,负责接收前台的请求,完成对镜像,容器,仓库等的各种操作

DockerImage:Docker镜像仓库

DockerContainer:Docker容器

DockerRegistry:Docker仓库

因此访问的时候是Docker的这个守护进程在主机上运行,然后通过Socket连接从客户端访问,守护进程从客户端接收命令并且管理运行在主机上的容器

文件挂载的原理是什么?

挂载其实就是连接,它说的是将设备文件中的顶级目录连接到Linux根目录下的某一个目录,访问这个目录就相当于访问设备文件

同一台机器上的所有容器都共享主机操作系统的内核,有了容器镜像之后,容器就解决了一致性的问题,由于镜像中打包的不仅仅是应用,而是整个操作系统的文件和目录,这也就意味着应用以及它允许所需要的依赖,都被封装在了一起,这种深入到操作系统级别的运行环境的一致性,打通了应用在本地开发和云端执行环境之间难以逾越的鸿沟

Docker隔离性不好体现在哪里?

  • 由于容器之间是共享内核资源的,因此当有一个容器占用资源过多,其他容器就无法使用资源了
  • 很多名录都是通过proc、sys等目录来读取信息的,比如说top、free等都是通过这个目录来计算资源的,常见的文件有/proc/meminfo、/proc/devices/system/cpu等文件,由于容器中没有实现对这些文件的隔离,因此最终就会导致问题的发生

Docker的网桥模式是怎么样的?

通过namespace可以知道每个容器都有一个独立的NetWork空间,以网桥模式为例,除了分配隔离的网络命名空间之外,Docker还会为所有的容器设置IP地址,当Docker服务器在主机上启动之后,就会创建新的虚拟网桥docker0,随后在该主机上启动的全部服务在默认情况下都与该网桥相连的

在默认情况下,每一个容器在创建的时候都会创建一对虚拟网卡,两个虚拟网卡组成了数据的通道,其中一张网卡会放在创建的容器中,然后会加入到名为docker0的网桥中

docker0会为每一个容器分配一个新的IP地址,然后将docker0的IP地址设置为默认的网关,网桥docker0通过iptables中的配置与宿主机上的网卡相连,所有符合iptables的请求都会通过iptables转发到docker0,然后通过网桥分发给对应的机器

端口映射的原理是什么?

映射原理是在iptables添加一个路由,这个路由规定了从哪发过来的包要转发到哪里去

DNAT tcp -- anywhere anywhere tcp dpt:6379 to 192.168.0.4:6379

通过这个路由表,它就会将来自任意源发送到当前机器的6379端口的TCP包转发到192.168.0.4:6379的所在的地址上,这其实就相当于充当一个网关的作用,收到这个包,然后转发到本地的路由表上

当有Docker容器需要将服务暴露给宿主机服务器,就会为容器分配一个IP地址,然后向iptables中追加一条新的规则

libnetword

整个网络部分的功能都是通过Docker拆分出来的libnetwork实现的,它提供了一个连接不同容器的实现,同时也能够为应用给出一个能够提供一致编程接口和网络层抽象的容器网络模型

每一个容器中包含了一个sandbox,保存了当前网络中的网络栈配置,包括有接口,路由表,DNS设置等,Linux使用网络命名空间实现这个SandBox,每一个SandBox都可能会有一个或者多个EndPoint,在Linux上就是一个虚拟的网卡,SandBox通过EndPoint加入到对应的网络中,这里的网络就是对应的网桥或者VLAN

挂载是如何实现的?

MountNamespace:这个namespace用来保证容器所看到的文件系统的视图

在新的进程中创建隔离的挂载点的命名空间需要在clone()函数中插入CLONE_NEWS,这样的话子进程就能够得到父进程挂载点的拷贝。

如果一个容器需要启动,那么它肯定要提供一个根文件系统,容器需要使用这个文件系统来创建一个新的进程,所有的二进制执行程序都需要在这个根目录下执行

2. 没A出来的算法题

easy题,傻逼了,大脑一篇空白

注意做二叉树的题目的时候,首先要想明白怎么去遍历,不要着急上手写

迭代法的实现

我们想象一下这道题,首先就是模拟:

  • 什么时候加法?遍历到两颗树的相同的位置的时候,就将这两个节点的值加起来,由于我们最终返回的是左边那棵树的根节点,因此我们将值添加到左子树上
  • 什么时候做替换?当左边的左子树为空的时候,右边的左子树不为空的时候,就将左边的左子树替换为右子树
  • 什么时候做替换?当左边的左子树不为空的时候,右边的左子树为空,那么就直接走左子树
class Solution {
    public TreeNode mergeTrees(TreeNode root1, TreeNode root2) {
        //1. 特殊情况,减少时间
        if(root1 == null){
            return root2;
        }
        if(root2 == null){
            return root1;
        }
        TreeNode p = root1;
        //2.使用层序遍历的解法,每次都将左右子树压入队列中,我们最终返回root1
        Queue<TreeNode> q = new LinkedList<>();
        q.offer(root1);q.offer(root2);
        while(!q.isEmpty()){
            TreeNode node1 = q.poll();TreeNode node2 = q.poll();
            node1.val += node2.val;
            if(node1.left!= null && node2.left!= null){
                q.offer(node1.left);q.offer(node2.left);
            }if(node1.right != null && node2.right != null){
                q.offer(node1.right);q.offer(node2.right);
            }
            if(node1.left == null && node2.left != null){
                node1.left = node2.left;
            }
            if(node1.right == null && node2.right != null){
                node1.right = node2.right;
            }
        }
        return p;
    }
}

递归法的实现

class Solution {
    public TreeNode mergeTrees(TreeNode root1, TreeNode root2) {
        if(root1 == null){
            return root2;
        }
        if(root2 == null){
            return root1;
        }
        root1.val += root2.val;
        root1.left = mergeTrees(root1.left,root2.left);
        root1.right = mergeTrees(root1.right,root2.right);
        return root1;
    }
}

3. 复习二叉树的相关知识

二叉树的定义

一棵树中的任意两个节点有且只有一条路径连通

一棵树如果有n个节点,那么它一定有n-1条边,这个可以从规律的角度来看,当只有一个节点的时候,此时就没有边

当有两个节点的时候,就只有一条边,可以看出,从数学归纳法的角度上来看:

当n = 1的时候,边数 = 0

自此之后,每增加一个节点,就会多增加一条边,因此边数就是n-1

二叉树的高度是什么?

二叉树的高度是指节点距离叶子节点的所要经过的边数

二叉树的深度是指节点距离根节点所要经过的边数

二叉树的层数指的是从根节点开始,要向下遍历的次数+1,也就是深度+1

二叉树第i层至多有2^(i-1)个节点啊,深度为k的二叉树至多有2^(k+1)-1

树的高度如何计算?

其实计算的是,一个具有n个元素的数组,除于多少次2能够得到一个元素

比如说有n个节点,假设除于x次2就能够得到一个元素,也就是它其实是一棵完全二叉树

n/2^x = 1,也就是2^x = n,那么也就是x = log2n+1(向下取整),然后这个

什么是满二叉树?

满二叉树指的是每一层的节点数量都达到最大值的话,那么这个二叉树就值满二叉树

什么是完全二叉树

所谓完全二叉树就是说对于深度为K的,有n个节点的二叉树,当且仅当每一个节点都与当前深度为K的满二叉树中编号从1-n的节点一一对应的就叫做完全二叉树

平衡二叉树

平衡二叉树是一棵二叉排序树,它可以是一棵空树,如果不是空树,那么它的左右两个子树的高度差的绝对值不超过1,并且左右两个子树都是一棵平衡二叉树

4. 复习如何估算时间复杂度

时间复杂度是一个函数,它定性地描述了算法的运行时间

假设算法的输入规模为n,那么操作单元数量用函数f(n)来表示,随着数据规模n的增大,算法执行时间的增长率和 f(n)的增长率相同,这靠左算法的的渐进时间复杂度

什么是bigO表示法

bigO是用来表示上界的,当用它作为算法的最坏的运行情况运行时间的上界

时间复杂度低,就代表性能一定好吗?

我们说时间复杂度是在最坏的情况下的一个估算,注意它是一个估算,那么也就是说它会省略掉一些常数项

比如说O(n^2)的时间复杂度一定优于O(n),在数据量小的时候,不一定,比如说原本的式子是:

n^2,n+1000000,在数据量小的时候,就会导致一个O(n)的算法复杂度是更慢的

一般来说排序有:

O(1) < O(logn) < O(n) < O(nlogn) < O(n^2) < O(n ^3 )< O(2^n)

同时要注意大常数,当这个大常数达到万以上的级别的时候,就不能够忽略了

复杂表达式的化简?

  • 去掉常数项
  • 去掉常数系数
  • 保留最高项,去掉后面小一级的

O(logn)中的log是以什么为底的?

这个算法的时间复杂度是O(logn)的时候,那么一定是以2为底n的对数么?

其实不然,也可以是以10为底,n的对数,也可以是以20为底,n的对数

为什么能够忽略底数,这里的举个例子

比如说log2n和log10n吧,这两个时间复杂度要怎么比较?

根据高中数学:log2n = log210*log10n,因此这个写法就是省略掉了常数系数了

如何估算递归的算法复杂度?

递归的算法复杂度 = 递归的次数*每次递归中的操作次数

以归并排序为例,可以怎么说时间复杂度?

可以这样说,将递归树画出来,然后最顶层是n,最底层是1

因为递归的时间复杂度 = 递归的次数*每次递归中的操作次数

由于每次递归中的操作次数是不固定,因此我们细分到一个递归树的节点上是很难计算的

递归排序的时间 = 分解时间 + 排序时间的时间 + 合并的时间

因为每一层递归树中,每一层都是n个元素进行排序,那么一共有多少次合并呢?就是递归树的高度
